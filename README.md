## 1. Definição do Domínio

**Smart Hub** – Plataforma Integrada de Gestão para Condomínios-Clube.

### Contexto do Sistema
Condomínios de grande porte, funcionam como verdadeiras "mini cidades". Eles possuem centenas de unidades habitacionais e uma infraestrutura complexa que inclui diversos salões de festas, quadras poliesportivas, academias, spas, mercados internos e uma rede de prestadores de serviços (segurança, elétrica, mecânica). A gestão manual ou via grupos de mensagens desses serviços é ineficiente e propensa a erros.

### Problema que o sistema pretende resolver
Atualmente, moradores enfrentam dificuldades para verificar a disponibilidade de áreas comuns em tempo real, gerir o fluxo de visitantes e encontrar profissionais de manutenção confiáveis. Por outro lado, a administração sofre com a descentralização das informações e a falta de métricas sobre o uso das dependências e incidentes de segurança.

### Público-alvo
* **Moradores:** Buscam conveniência na reserva de espaços e contratação de serviços.
* **Administração (Síndicos e Zeladores):** Necessitam de controle centralizado e comunicação direta.
* **Portaria/Segurança:** Precisam de agilidade na liberação de acessos.
* **Prestadores de Serviços:** Buscam visibilidade e organização de chamados técnicos.

## 2. Organização do Projeto

Este projeto utiliza a metodologia ágil **Kanban** para a gestão de tarefas. Todas as funcionalidades foram mapeadas como **User Stories** (Histórias de Usuário) e estão organizadas no [GitHub Projects]

### Quadro Kanban
O fluxo de trabalho está dividido em:
* **Backlog:** Todas as histórias de usuário planejadas.
* **Todo:** Tarefas priorizadas para execução imediata.
* **In Progress:** Funcionalidades em fase de desenvolvimento.
* **Done:** Funcionalidades finalizadas e validadas.
